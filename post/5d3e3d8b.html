模拟力学仿真计算

  • 插件总结
    • Motion 使用完整的运动性建模来计算零部件运动
    • SimulationXpress：一个简单形变仿真的向导，可以显示单个零件受力后的形变过程
    • simulation：静应力分析，拓扑优化等
    • Flow Simulation：研究流体力学

操作步骤

  • 首先点击simulation 插件中的simulation premium，待标题栏出现simulation后点击图标
  • 接着点击新算例，一般选择静应力分析，
  • 先对应用材料信息进行设置（最好先在零件中完成设置）
    • 在零件编辑树中右键—材料—编辑材料，可自定义材料信息
      ![[Pasted image 20231020090729.png]]
  • 选择夹具顾问中的固定几何体，进而对零件某个面进行固定
  • 点击运行此算例中的运行网格，网格密度一般选择中间（网格越密效果越好）
  • 在外部载荷顾问中选择合适的力
  • 点击运行此算例，接着就可以在结果一栏中看到分析结果
    • 有应变，位移应变分析,下图是放大了4000倍
      ![[Pasted image 20231020095505.png]]
  • 最终结果是看实际机械结构的最大应力，然后仿真结果进行对比，看是否超出范围

  • 在装配装配体前首先设置好原点，这样可以保证初始的坐标系
    • 也就是装配体插入第一个零件时，先设置成浮动，接着使原点和零件中的原点重合
    • 最后配合好的形状也就是urdf和gazebo中的初始状态模型
    • 注意base_link的坐标要设置在与地面接触的地方，若在机身中会在gazebo中零件会不规律，报错
  • 首先先安装urdf插件
  • 对于模型需要设置坐标系的零件来说，首先先设置好基准轴(旋转轴)
    • 插入—参考几何体—基准轴、坐标系
    • 由于自动生成的坐标系可能有问题，所以说需要我们自己去设置基准轴，坐标系等
    • 思路，先设置出基准轴，接着根据轴与平面(或者其他轴)相交产生一个点,根据这个点生成坐标系并调整。
      • 对于坐标系来说，需要限定两个方向才能将这个坐标系固定

    • join标签详解
    • 添加的零件是绕着该旋转轴直接旋转的关节，最好添加完整，否则只会显示添加的零件，也就是让这个轴旋转的所有关节b，但b(b包含a)不包含以b为父关节的子关节a
    • Link代表骨骼，就是每一个部分。joint代表关节，就是两个关节的组合(以腿为例：Link_小腿，Link_大腿，组合成joint_膝盖)

urdf文件内容详解

  • 手写移动机器人urdf详解

  • URDF各参数设置备忘

  • urdf中坐标系的对应关系是，当前joint坐标系是对上一个joint坐标系的变换，当前inertial坐标系是相对于当前joint坐标系的变化（是重心相对于当前坐标系的距离，urdf的惯性张量是相对于重心的惯性张量），当前visual坐标系是相对于当前joint坐标系的变换，当前collision坐标系是相对于当前joint坐标系的变换。

  • 这里的rpy变换一定要注意，是先按当前z轴（即上一级joint坐标系的z轴）进行旋转（rpy中y的值），然后绕变换以后的坐标系y轴进行变换（rpy中p的值），最后绕上一次绕y轴变换之后坐标系的x轴进行变换（rpy中r的值）。所以urdf的rpy变换，属于intrinsic范畴，按矩阵连乘表示，按照顺序——Z，Y，X

使用link标签创建模型

  • link标签是用以描述机器人某个刚体部分的外观和物理属性的，其参数包括连杆尺寸（size）、颜色（color）、形状（shape）、惯性矩阵（inertial matrix）、碰撞参数（collision properties）等。
  • <inertial>：描述了惯性信息，包括质量、质心位置和惯性矩阵。
    • <origin>：描述质心的位置和姿态。xyz属性是质心的位置，rpy是质心的姿态
    • <mass>：描述链接的质量。
    • <inertia>：描述链接的惯性矩阵。
  • <visual>：描述了链接的可视化外观。
    • <origin>：描述可视化外观的位置和姿态。
    • <geometry>：描述可视化外观的几何形状。
      • <mesh>：描述链接的网格形状，其中”filename”属性指定了网格文件的路径。
    • <material>：描述了材质的外观属性。
      • name属性：指定了材质的名称。
      • <color>：描述了材质的颜色属性。
      • rgba属性：RGBA（红、绿、蓝、透明度）值通常在0到1之间取值。每个通道的取值范围是从0到1，其中0表示最小的颜色强度或透明度
  • <collision>：描述链接的碰撞检测外观，用于物理交互模拟。
    • <origin>：描述碰撞检测外观的位置和姿态。
    • <geometry>：描述碰撞检测外观的几何形状。
      • <mesh>：描述链接的网格形状，其中”filename”属性指定了网格文件的路径。

joint标签连接模型

  • <joint>：描述了一个关节。
    • name属性：指定了关节的名称，这里是”right_hind_joint5”。
    • type属性：指定了关节的类型，这里是”continuous”，表示关节可以无限制地旋转。
  • <origin>：描述了关节原点的位置和姿态。
    • xyz属性：指定了关节原点的位置
    • rpy属性：指定了关节原点的姿态
  • <parent>：描述了该关节的父链接（parent link）。
    • link属性：指定了父链接的名称，这里是”right_hind_link4”
  • <child>：描述了该关节的子链接（child link）
    • link属性：指定了子链接的名称，这里是”right_hind_link5”
  • <axis>：描述了关节的旋转轴
    • xyz属性：指定了旋转轴的方向，这里是(0, 0, -1)，表示绕Z轴逆时针旋转。

sdf文件和urdf和xacro文件的区别

  • SDF文件用于描述仿真环境，URDF文件用于描述机器人模型，而Xacro则是一种用于简化URDF文件编写的工具
  • sdf文件、urdf文件和xacro文件都是模型文件。
  • xacro文件是urdf文件的改进版，urdf文件只能在rviz等中显示，不能在仿真器中显示出来。
  • xacro文件可以在gazebo仿真器中显示出来，相对urdf文件，xacro文件增加了更多的属性设置标签。
  • sdf文件和urdf文件、xacro文件都可以加载dae等三维模型文件。
  • sdf和urdf、xacro的区别
    • ros和gazebo 不能使得sdf模型动起来、sdf模型有自己的行为
    • ros和gazebo能使得urdf、xacro模型动起来、urdf、xacro模型有自己的行为
  • 当前，sdf文件和urdf、xacro文件之间还不能自动转换，只能根据实体手动转换。

添加gazebo控制器插件

  • <plugin>:插件元素，用于在Gazebo中加载指定的插件。
    • name=”gazebo_ros_control”: 插件的名称，这里是”gazebo_ros_control”。
    • filename=”libgazebo_ros_control.so”: 插件的文件名，这里是”libgazebo_ros_control.so”。这表示Gazebo将加载名为”libgazebo_ros_control.so”的共享库文件作为该插件。
    • 插件的名称可以是任意的，但需要确保插件名称与加载的插件文件名匹配。在上述代码中，插件的名称是”gazebo_ros_control”，并且加载的插件文件名是”libgazebo_ros_control.so”。这是因为在加载插件时，Gazebo会查找与插件名称匹配的共享库文件。
  • <robotNamespace>/var_strc_uav</robotNamespace>: 机器人命名空间。这表示该插件将在名为”/var_strc_uav”的命名空间下加载。也就是文件第二行机器人的命名名称：robot name="var_strc_uav"，也就是后面话题会用到
  • <robotSimType>gazebo_ros_control/DefaultRobotHWSim</robotSimType>: 机器人仿真类型。这表示Gazebo将使用”gazebo_ros_control/DefaultRobotHWSim”作为机器人的仿真类型。
  • true: 是否启用遗留模式的命名空间。这里设置为”true”，表示启用遗留模式命名空间。
